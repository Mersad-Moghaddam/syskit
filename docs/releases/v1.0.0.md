# SysKit v1.0.0

SysKit v1.0.0 is the first stable release of the Linux-native system inspection,
monitoring, and diagnostics toolkit.

## Highlights

- Native `/proc`, `/sys`, Netlink, and cgroup collection without wrapping system
  utilities.
- System, CPU, memory, disk, filesystem, process, network, port, container, and
  explainable diagnostic views.
- Interactive dashboard, top, and generic watch modes.
- Deterministic table, JSON, and YAML output with a test-enforced v1 contract.
- Explicit bounded out-of-process plugin protocol v1.
- Reproducible amd64/arm64 archives, deb/rpm packages, AUR metadata,
  `SHA256SUMS`, and an installable `syskit(1)` manual.

## Compatibility

The command, flag, configuration, exit-code, plugin, and structured-output
surfaces in the [v1 compatibility contract](../compatibility.md) are stable under
Semantic Versioning. Scripts should consume JSON or YAML rather than table
spacing.

## Release Validation

- Formatting and import grouping clean.
- `go vet`, build, race tests, Linux integration tests, golden tests, contract
  tests, and benchmarks pass.
- GitHub CI and its `govulncheck` stage pass on the release commit lineage.
- amd64/arm64 archives and Debian/RPM/AUR layouts were built and inspected.
- Repeated builds with a fixed `SOURCE_DATE_EPOCH` produced byte-identical
  archives and packages.
- Markdown relative links, required documentation artifacts, and `syskit(1)`
  rendering were checked.

## Known Limitations

- Disk saturation is reported unavailable because SysKit does not yet collect
  device busy-time utilization.
- Container data is cgroup-derived; runtime names and lifecycle status are not
  queried from runtime sockets.
- Release artifacts have SHA-256 checksums but are not cryptographically signed.
- Interactive views require a TTY.
